How to download and add local models using Ollama. Config changes need a server restart to take effect.

1 — Find a model

Browse ollama.com/library for available models. Recommended for tool use:

llama3.2:3bFast · low RAM · good for Pi / constrained laptops
llama3.1:8bBalanced · best quality/speed on 16 GB machines
qwen2.5:7bStrong tool calling · recommended for MCP workflows
mistral:7bGood structured output · reasoning tasks
qwen2.5:14bHigher quality · needs 16 GB+ RAM
qwen2.5:32bBest local quality · needs 32 GB+ RAM
2 — Pull the model

Run this in your terminal. Ollama must be running.

ollama pull llama3.2:3b

To list what you already have locally:

ollama list
3 — Add it to config.local.yaml

Open the Config File tab and add an entry under models::

models:
  - display_name: "My Model Name"
    ollama_model: "llama3.2:3b"
    description: "Optional description shown in UI"

The display_name is what appears in the model selector. Set default_model under agent: to the ollama_model value to make it the default on load.

Import a GGUF model (e.g. from HuggingFace)

Download a .gguf file, then create a Modelfile and import it:

echo "FROM /path/to/model.gguf" > Modelfile
ollama create my-model -f Modelfile

Then add my-model as the ollama_model value in config.local.yaml.

Hardware guidance
Raspberry Pi 5 (8 GB)3B models only
MacBook Air M1 (16 GB)Up to 8B comfortable
Mac Mini M4 (16 GB)Up to 13B fine, 30B feasible
32 GB+30B–70B quantised models
